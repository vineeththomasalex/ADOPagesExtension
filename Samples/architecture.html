Components

API Gateway

All external traffic enters through the API Gateway, which handles:

  • Authentication — JWT validation via OIDC identity provider
  • Rate limiting — 120 requests/min per token (sliding window)
  • TLS termination — Certificates managed by Azure Front Door
  • Routing — Path-based routing to backend service clusters

Widget Service

Core business logic running as a containerised app on Kubernetes:

  • Validates inputs, enforces business rules, publishes domain events
  • Runs 3 replicas behind an internal load balancer
  • Communicates with the gateway via gRPC (Protobuf)
  • Read-heavy queries hit the Redis cache first; writes go straight to PostgreSQL

Data Stores

StoreRoleConfiguration
PostgreSQL Source of truth for widget data Primary + 1 read replica, daily backups, 30-day retention
Redis Query cache & session store Cluster mode, 6 GB, 15-min TTL on cache entries

Event Bus & Workers

Domain events (e.g. widget.created, widget.archived) are published to Azure Service Bus. Downstream consumers include:

  • Notification Worker — Sends email/Teams alerts on widget changes
  • Audit Logger — Writes immutable audit records to cold storage
  • Search Indexer — Updates the full-text search index
