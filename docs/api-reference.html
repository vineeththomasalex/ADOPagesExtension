Widget Service — API Reference

Internal REST API for managing widgets · v2.4

Overview

The Widget Service exposes a RESTful API over HTTPS. All endpoints accept and return JSON. Authentication is via Bearer token in the Authorization header.

Base URL:

https://api.example.com/v2

Authentication

Include a valid token with every request:

GET /widgets HTTP/1.1
Host: api.example.com
Authorization: Bearer <token>

Tokens expire after 60 minutes. Use the /auth/refresh endpoint to obtain a new token.

Endpoints

GET /widgets

List all widgets, with optional filtering and pagination.

Query Parameters

ParamTypeDefaultDescription
pageinteger1Page number
per_pageinteger25Items per page (max 100)
statusstring—Filter by status: active, archived
searchstring—Full-text search on name & description

Example Response

{
  "data": [
    {
      "id": "w-8a3f",
      "name": "Sprocket Alpha",
      "status": "active",
      "created_at": "2025-11-02T14:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 25,
    "total": 142
  }
}

POST /widgets

Create a new widget.

Request Body

FieldTypeRequiredDescription
namestringYesDisplay name (3–120 chars)
descriptionstringNoMarkdown-formatted description
tagsstring[]NoCategorization tags

Example Request

POST /widgets HTTP/1.1
Content-Type: application/json

{
  "name": "Sprocket Beta",
  "description": "Next-gen sprocket with **turbo** mode.",
  "tags": ["hardware", "beta"]
}

Example Response 201 Created

{
  "id": "w-9b4g",
  "name": "Sprocket Beta",
  "status": "active",
  "created_at": "2025-12-10T09:15:00Z"
}

GET /widgets/:id

Retrieve a single widget by its ID.

Example Response

{
  "id": "w-8a3f",
  "name": "Sprocket Alpha",
  "status": "active",
  "description": "The original sprocket.",
  "tags": ["hardware", "stable"],
  "created_at": "2025-11-02T14:30:00Z",
  "updated_at": "2025-12-01T08:00:00Z"
}

PUT /widgets/:id

Update an existing widget. Supports partial updates.

PUT /widgets/w-8a3f HTTP/1.1
Content-Type: application/json

{
  "description": "The original sprocket — now with improved durability."
}

DELETE /widgets/:id

Archive a widget. Returns 204 No Content on success.

Error Handling

All errors follow a consistent shape:

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Field 'name' is required.",
    "details": []
  }
}
HTTP StatusCodeMeaning
400VALIDATION_ERRORInvalid request body or parameters
401UNAUTHORIZEDMissing or expired token
403FORBIDDENInsufficient permissions
404NOT_FOUNDResource does not exist
429RATE_LIMITEDToo many requests (limit: 120/min)
500INTERNAL_ERRORUnexpected server error

Rate Limiting

The API enforces a rate limit of 120 requests per minute per token. Rate-limit headers are included in every response:

HeaderDescription
X-RateLimit-LimitMaximum requests per window
X-RateLimit-RemainingRequests remaining in current window
X-RateLimit-ResetUTC epoch seconds when the window resets
